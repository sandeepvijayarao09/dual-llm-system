Project Contributions

🔀

Three-Layer Cascade

Pre-flight + offline TF-IDF ML router + LLM-based JSON routing fallback — 85.3% handled at zero API cost.

🪄

Personalization-by-Compression

Logistic Regression Pipeline
  • Features: 16 hand-crafted tag signals + TF-IDF weights.
  • Reasoning triggers: 'why', 'derive', 'analyze', 'prove'.
  • TF-IDF Vectorization: 1–3 ngrams, 5,000 features.
  • Model serialized via Joblib for < 5 ms latency.
  • Never worse than an LLM-only baseline — conservative fallback.

Each tag fires when its pattern matches and is prepended to the raw query before TF-IDF encoding.

LEN_MEDIUM9–40 wordsneutral
LEN_LONG> 40 words→ big

HAS_FACTUAL_KWwhat is, who is, when did, define…→ small

Personalization-by-Compression

🧠
Reason
Large LLM writes deep core answer.
No user profile.
📝
Summarize
Small LLM compresses core to one sentence (≤80 tokens).
👤
Add Context
Small LLM injects user profile — intro + closing wrapper.
✅
Assemble
Final output:
intro + core + "---" + closing
🔒 Privacy Win: The Large LLM never sees the user profile. Profile context only enters through the Small LLM wrapper — limiting exposure of personal data to the most powerful model.
💰 Cost Win: Large LLM output is never re-injected into context windows. One-sentence summary prevents exponential token growth across the personalization chain.
Profile injection is conditional: the LLM classifier's profile_relevant flag decides per-query. Self-referential queries ("what is my name") always inject — hard override.

Bounded Conversational Memory

The ConversationBuffer
  • Keeps last N = 3 turn-pairs verbatim in context.
  • Older turns folded into a rolling summary (≤ 80 tokens) via Small LLM.
  • Amortized O(1) processing cost per turn.
  • Assistant turns truncated to 400 characters before buffering.
  • No-LLM fallback: if summarization fails, truncation-only summary used.
Context sent to model each turn:
[System]: "Earlier: Alex asked about recursion, REST APIs…"
[User/Asst]: turns N-2, N-1, N verbatim
[User]: current query

Summary & Future Work

Limitations
  • Small held-out set used in v0 (N=22); v1 uses 1,000-case benchmark.
  • Binary labels only (Small vs Big) — fine-grained routing not yet supported.
  • Uncalibrated τ threshold (0.65 set empirically, not via held-out optimization).
  • Residual profile leakage — name stub still sent on impersonal queries.
  • Relies on proprietary OpenAI API — model deprecation risk.
  • debug_simple at 33.3% — next immediate fix.
Next Steps
  • Add HAS_DEBUG_SIMPLE tag + 30 seed examples → fix debug_simple.
  • Export agreement-only logs from Router v1 → retrain Router v2.
  • Expand to 4+ routing classes (e.g., code, math, reasoning, factual).
  • Online A/B testing against LLM-only baseline with real users.
  • Port to open-weight models (Llama-3, Mistral) for API-free deployment.
  • Energy-efficient edge hardware deployment (Raspberry Pi / mobile).
